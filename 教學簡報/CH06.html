機器人課的招牌時刻

這章 OTTO 要學會向前走、向後退，而且你能控制它走幾步、走多快。從此它不再只是原地表演——整個 3D 舞台都是它的。

🎁 本章小成品

OTTO 不停巡邏：向前 4 步 → 後退 2 步 → 唱歌 → 休息 → 再來。而且每個數字你都改得動。

新指令：Otto.walk 的三個參數

括號裡三個數字，各管一件事
Otto.walk( 步數, 速度 T, 方向 );
Otto.walk(  4,   1000,   1  );   // 向前走 4 步
參數意思可以填什麼
① 步數走幾步1、2、4、10……你決定
② 速度 T一步花幾毫秒600（快）～ 1400（慢），數字越大越慢
③ 方向往哪走1 = 向前，-1 = 向後
🤔 「速度數字越大越慢」很反直覺？因為 T 是「一步花的時間」——時間越長，當然走越慢。

逐段看懂：loop() 裡的巡邏

每個數字都有意義
void loop() {
  Otto.walk(4, 1000, 1);   // 向前走 4 步（每步 1000 毫秒）
  Otto.walk(2, 1000, -1);  // 向後退 2 步（-1 就是倒退）
  Otto.sing(S_happy);      // 唱一段開心的音
  delay(1000);             // 休息 1 秒再重來
}

試著在心裡「跑」一遍這段程式，預測 OTTO 的動作順序——先預測、再執行、對答案，這是看懂程式最快的方法。

🔬 小實驗：速度的手感

直接在模擬器改數字，不用回生成器
  • 在模擬器程式面板，把第一行的 1000 改成 600
  • 按 ▶ 執行——是不是變快了？
  • 再改成 1400 執行——慢動作！
💡 直接改數字再執行，是最快的實驗法；大改動（加動作、改邏輯）再回去找 AI。兩招搭配就是高效率的 Loop 工程。

🔁 迭代任務

設計你的巡邏路線

拿著程式找 AI 迭代（①說需求 ②拿新程式 ③模擬器驗證 ④回饋修正）：

  • ⭐ 改成向前 6 步、不後退
  • ⭐⭐ 走路又快又遠：前進 8 步、速度 700，唱 S_superHappy
  • ⭐⭐⭐ 來回等距巡邏：前進 4 步、後退 4 步——執行後觀察：OTTO 是不是在原地來回？為什麼？
💬 回饋句型：「現在 OTTO 前進 4 步後退 2 步，我想改成 ______，請給我完整程式。」

常見問題

卡住先看這

OTTO 越走越遠不見了？

⟲ 歸位。想讓它在附近活動，前進後退步數設一樣。

紅字「看不懂這一行」？

八成是括號裡忘了逗號、或分號不見了。整段紅字貼給 AI。

速度 300 可以嗎？

模擬器跑得動，但真機馬達會抖到站不穩——600~1400 是真機安全範圍，養成好習慣。

想走「斜的」？

walk 只會直線。轉方向要用 turn——下一章登場！
