Arduino IDE：讓「你改造的程式」也能上真機

這一章要做出什麼

解鎖完全自由

「一鍵上傳」只能傳老師的標準程式。這章學 Arduino IDE——真正的上傳工具，從此任何程式（包括你 CH10 的舞蹈秀、迭代任務的改造版）都能親手送進真的 OTTO。

🎁 本章小成品

你自己改造的程式，在真正的 OTTO 身上跑起來。這是這門課最有成就感的一刻。

📋 事前檢查：CH02 的 CH340 驅動裝了嗎？（裝置管理員看得到 USB-SERIAL CH340）沒裝先回 CH02。
步驟 1 / 7

安裝 Arduino IDE

安裝檔在課程資料夾，不用上網載
  • 打開課程資料夾 工具下載，找到 arduino-1.8.12-windows.exe，點兩下
  • 「是否允許變更」→ 是
  • 「License Agreement」→ I Agree
  • 元件選擇不用改 → Next → 安裝位置不用改 → Install
  • 跑完出現 Completed → Close；過程若跳出「要不要安裝裝置軟體」→ 都按安裝
✅ 預期：桌面出現水藍色「Arduino」圖示。

❓ 為什麼用 1.8.12 不用最新版？這是原廠指定版本，和 OTTO 函式庫最合、問題最少。

步驟 2 / 7

安裝 Otto9 函式庫

把「能力包」放進指定資料夾
  • 到 工具下載 找 Libraries_V9.zip → 按右鍵 → 全部解壓縮
  • 解壓後打開，會看到 Libraries 資料夾，裡面有 Otto9、Oscillator、US… 一堆資料夾
  • 全選這些資料夾（Ctrl+A）→ 複製（Ctrl+C）
  • 打開 文件（Documents）→ Arduino → libraries → 貼上（Ctrl+V）
✅ 預期：libraries 資料夾裡出現 Otto9 等十來個資料夾。

❓ 找不到「文件\Arduino\libraries」？先開一次 Arduino IDE 再關掉，它會自動建好。

步驟 3 / 7

放入你的程式

從模擬器到 IDE
  • 打開 Arduino IDE（水藍圖示）
  • 把畫面裡的預設程式 全選刪掉（Ctrl+A → Delete）
  • 貼上你要上傳的程式（模擬器驗證過的那份，Ctrl+V）
  • 存檔 Ctrl+S → 取個名字（例如 my_show）→ 儲存
💡 黃金原則：先模擬、後上傳——只上傳「模擬器跑過沒問題」的程式，真機一次就成功。
步驟 4 / 7

選開發板

告訴 IDE：我的機器人是 Nano
  • 上方選單 工具（Tools）→ 開發板 → Arduino AVR Boards → Arduino Nano
  • 工具 → 處理器 → ATmega328P (Old Bootloader)　← 我們的套件用這個！
✅ 預期：視窗右下角顯示「Arduino Nano, ATmega328P (Old Bootloader)」。

❓ 之後上傳若一直失敗，回這裡把處理器換成「ATmega328P」（不帶 Old）再試——兩種晶片版本都存在。

步驟 5 / 7

接線、選序列埠

跟一鍵上傳一樣的線
  • USB 傳輸線接「電腦 ⇄ OTTO 頭後方」
  • 工具 → 序列埠 → COM3（或其他數字）——就是裝置管理員看到的那個 CH340
✅ 預期：序列埠選單有 COM 可選、被打勾。

❓ 序列埠灰灰的不能選？線沒插好、或驅動沒裝（回 CH02）、或線是純充電線。

步驟 6 / 7

上傳！

按下那顆 → 箭頭
  • 按工具列的 →（右箭頭）上傳按鈕
  • 下方黑色區域開始跑字：先「編譯中…」再「上傳中…」（OTTO 板上的燈會狂閃＝正在寫入）
  • 等到出現「上傳完成」
✅ 預期：狀態列顯示「上傳完成」，OTTO 可能當場就開始動！
⚠️ 上傳時 OTTO 手腳可能亂揮——正常。拿在手上或放空曠處，別讓它掉桌。
步驟 7 / 7

驗收與換程式

從此你自由了
  • 拔線 → 開 OTTO 電源 → 看它跑你的程式
  • 想改？回模擬器迭代 → 改好貼回 IDE → 再按上傳（會覆蓋舊的）
🏆 里程碑達成

你已打通全流程：AI 生成 → 模擬驗證 → 迭代修正 → 真機上傳。這就是完整的 Vibe Coding × Loop 工程。

常見問題（上傳失敗急救包）

90% 的問題在這裡解決

avrdude: stk500_getsync() 錯誤？

最常見！①處理器換成另一種（Old ↔ 非 Old）②序列埠選對了嗎 ③換 USB 孔、換線。

編譯錯誤 Otto9.h: No such file？

函式庫沒裝好，回步驟 2 重做，確認資料夾直接放在 libraries 下。

上傳完成但 OTTO 不動？

電源開了嗎？電池有電嗎？程式的動作是不是都在 loop() 裡？

錯誤訊息看不懂？

老招：整段複製貼給 AI——「Arduino 上傳出現這個錯誤，怎麼辦？」
