🎁 本章小成品

OTTO 沿正方形繞圈巡邏：直走 → 左轉 → 直走 → 左轉……走完一圈唱歌慶祝。從上往下看最清楚！

新指令：Otto.turn

跟 walk 長得很像，好記
Otto.turn( 步數, 速度 T, 方向 );
Otto.turn(  3,   1000,   1  );   // 原地左轉 3 步
參數意思可以填什麼
① 步數轉幾步（步數越多轉越多度）轉 90 度 ≈ 3 步（模擬器實測）
② 速度 T一步幾毫秒600 ~ 1400
③ 方向往哪轉1 = 左轉，-1 = 右轉
🧭 「轉 90 度要幾步」在真機上會因場地略有不同——這就是 CH16 校正章要處理的「模擬與現實的差距」。先記住：模擬器裡 3 步 ≈ 90 度。

路線 = 積木組合

正方形的「配方」

正方形怎麼走？「直走一段 → 左轉 90 度」重複 4 次。翻成 OTTO 語言：

直走 3 步   → Otto.walk(3, 1000, 1);
左轉 90 度 → Otto.turn(3, 1000, 1);
……以上兩行，重複 4 次 = 正方形！

這章我們先「土法煉鋼」把 4 個邊都寫出來（8 行）。你會覺得好重複、好累——記住這個感覺，CH11 的 for 迴圈就是來救你的。

逐段看懂：重複的痛苦

8 行裡有 6 行長一樣
void loop() {
  Otto.walk(3, 1000, 1);   // 第 1 邊
  Otto.turn(3, 1000, 1);   // 左轉
  Otto.walk(3, 1000, 1);   // 第 2 邊
  Otto.turn(3, 1000, 1);   // 左轉
  Otto.walk(3, 1000, 1);   // 第 3 邊
  Otto.turn(3, 1000, 1);   // 左轉
  Otto.walk(3, 1000, 1);   // 第 4 邊
  Otto.turn(3, 1000, 1);   // 左轉，回到起點方向
  Otto.sing(S_superHappy); // 慶祝！
  delay(2000);
}

看出來了嗎？同樣兩行重複了 4 次。想改邊長還得改 4 個地方，好麻煩……先忍耐，CH11 的 for 迴圈會讓這 8 行變 4 行。

🔁 迭代任務

設計你的專屬路線

找 AI 迭代（記得：說需求 → 驗證 → 回饋）：

  • ⭐ 正方形改右轉版（哪個參數要動？）
  • ⭐⭐ 邊長加倍：每邊 6 步——執行後從俯視角比較大小
  • ⭐⭐⭐ 挑戰三角形：三個邊、每個角要轉 120 度（提示：90 度≈3 步，那 120 度大約幾步？跟 AI 討論、實測微調——這正是 Loop 工程！）
💬 回饋句型：「轉彎角度好像不夠 / 太多，請把 turn 的步數改成 ______ 再給我完整程式。」

常見問題

卡住先看這

轉彎方向跟我想的相反？

從 OTTO 自己的角度看：1 = 它的左邊。你從正面看會相反——轉到它背後看就一致了。

越繞越偏離原點？

正常的累積誤差。按 ⟲ 歸位重來即可。

可以邊走邊唱嗎？

不行，指令是一個做完換下一個（這叫「循序執行」）。唱歌會在走完後發生。

程式好長好花？

對！這就是本章埋的伏筆——CH11 的迴圈讓它縮短一半以上。
