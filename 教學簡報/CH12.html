進入「會想」篇！

之前的 OTTO 只會照劇本演。從這章起它有了感官：超音波眼睛量出前方距離，OTTO 根據「看到的」自己決定怎麼反應——太近就驚叫後退、有東西就警戒、淨空就輕快哼歌。

🎁 本章小成品

OTTO 變成「距離報告員」：你拉模擬器的距離滑桿，它用不同的叫聲即時回報遠近。

超音波怎麼「看」？

跟蝙蝠一樣：聽回音
  • 左眼（Trigger，腳位 8）發出一聲人耳聽不到的超音波
  • 聲波撞到前面的東西，彈回來
  • 右眼（Echo，腳位 9）接住回音
  • 大腦算：「回音花了多久？」→ 時間 × 音速 ÷ 2 ＝ 距離
👀 在模擬器裡，每次量距離時 OTTO 的眼睛會閃紅光——那就是「正在發射超音波」的瞬間！執行後盯著看。

新概念：變數＋getDistance

量到的數字要有地方放
float distance = Otto.getDistance();
//  ↑類型   ↑名字      ↑量距離（回傳公分數）

變數就是「幫資料取名字的盒子」：float distance 準備一個能裝小數的盒子叫 distance，把 Otto.getDistance() 量到的公分數放進去。之後程式裡寫 distance 就是在用這個數字。

if (distance < 10) { ... 太近的反應 ... }
else if (distance < 25) { ... 中距離的反應 ... }
else { ... 淨空的反應 ... }

if / else if / else = 「如果…不然如果…再不然…」，三選一，只會走其中一條路。

模擬器執行：玩滑桿！

這章的主角是右上角的距離滑桿
  • 貼上程式 → ▶ 執行（聲音打開）
  • 把右上角「📡 超音波距離」滑桿拉到 30 以上 → 聽到輕快短音
  • 拉到 15 左右 → 變成「喔喔」警戒音
  • 拉到 8 以下 → 驚叫＋後退一步！
  • 同時觀察：眼睛每秒閃一次紅光、執行紀錄顯示「❓ if (distance < 10) → 成立/不成立」
✅ 預期結果：三段距離、三種反應，即時切換。你拉滑桿＝移動障礙物，OTTO 真的「看得到」。
