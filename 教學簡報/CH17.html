整學期本事的總驗收

設計一支「有個性的 OTTO」程式，在真機上發表。必須包含：

項目要求對應章節
🎬 開場秀音效＋動作亮相（setup 內）CH05、CH08、CH10
💃 表演元素至少 3 種舞步或手勢CH08、CH09
🔁 迴圈運用至少一個 for（重複或漸變）CH11

🎤 發表1 分鐘介紹：我的 OTTO 是什麼個性、哪段最得意、卡關怎麼解—

老師的整合範本

起點不是白紙——是骨架

生成器提供一支整合範本 otto_final.ino：開場秀 → 智慧巡邏（避障＋每 10 步慶祝一次）→ 全套結構都在。你的任務：以它為底，改造出你的版本。

  • 先生成範本（4 個目標：全域 → setup 開場秀 → loop 巡邏 → 偵測函式）
  • 模擬器跑一遍，看懂每一段
  • 列出你要改造的 3 個地方（開場？慶祝動作？避障反應？）
  • 跟 AI 迭代到滿意 → 模擬器驗證 → Arduino IDE 上傳真機
🚀 開啟整合範本生成器（新分頁）

🛟 救援通道

進度落後？沒關係，這裡接你上車
狀況救援方案
自己的程式改壞了回生成器重新生成標準版，從乾淨的重新改
來不及做出客製版一鍵上傳頁燒「otto_final.ino 標準版」，發表時介紹你理解的部分
真機上傳一直失敗先用模擬器發表（投影出來一樣精彩），課後找老師處理硬體
模擬器 OK 真機怪怪的檢查 CH16 校正做了沒；速度 T 提高到 1000 以上再試
🔌 一鍵上傳（救援用標準版都在裡面）

發表流程與評分

怎麼上台、看什麼
  • 介紹（30 秒）：我的 OTTO 叫什麼、什麼個性
  • 實演（60 秒）：真機表演——開場秀 → 巡邏互動（拿手掌當障礙物）
  • 心得（30 秒）：最得意的一段程式＋印象最深的一次 debug
面向比重看什麼
完成度40%規格四項有沒有到位、能不能跑
創意個性30%看得出「你的」設計，不只是範本
Loop 工程歷程20%說得出迭代過程：卡在哪、怎麼修好
表達10%介紹清楚、時間掌控
