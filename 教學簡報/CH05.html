新指令：playGesture 與 delay

這章會用到的兩個新朋友
指令意思例子
Otto.playGesture(手勢名)表演「動作＋音效」組合Otto.playGesture(OttoHappy);
delay(毫秒)暫停一下（1000 = 1 秒）delay(3000); 休息 3 秒

常用手勢名（共 13 種，先認識這幾個）：OttoHappy 開心、OttoWave 揮手、OttoSuperHappy 超開心、OttoSad 難過、OttoVictory 勝利、OttoSleeping 想睡。

💡 手勢名都是「Otto＋心情英文」，AI 都認得，跟它說中文心情它就會換。

❓ 只表演一次就停了？檢查那三行是不是貼進了 setup() 而不是 loop()——setup 只跑一次！

逐段看懂：loop() 在做什麼

三行程式 × 無限重複
void loop() {
  Otto.playGesture(OttoHappy);  // 開心搖擺（動作＋音效一起來）
  Otto.playGesture(OttoWave);   // 揮手打招呼（抖抖腿）
  delay(3000);                  // 休息 3 秒（3000 毫秒）
  // 跑到這裡，loop() 結束……然後自動再從第一行開始！
}

重點只有一個：loop() 的大括號 { } 裡面的事，會由上到下做完，然後回到第一行再來一遍。所以 OTTO 才會「每 3 秒打一次招呼」。

🔁 迭代任務

改造成「你的」打招呼

把完整程式貼給 AI，一步一步來：①說要改什麼 ②拿到新程式 ③貼模擬器驗證 ④不對就回饋再改。挑戰任選：

  • ⭐ 揮手改成勝利手勢（提示：OttoVictory）
  • ⭐⭐ 休息時間改成 5 秒（幾毫秒？自己算！）
  • ⭐⭐⭐ 打招呼前先唱一段 S_cuddly（提示：Otto.sing 要加在 loop 的哪一行？）
💬 回饋句型：「這支程式會每 3 秒打招呼，請把 ______ 改成 ______，其他不要動，給我完整程式。」

常見問題

卡住先看這

手勢表演到一半想停？

按 ⏹ 停止；OTTO 姿勢歪了就按 ⟲ 歸位。

紅字「不認識的名稱：OttoHapy」？

拼錯了！手勢名一個字母都不能差。整段紅字貼給 AI 它會幫你抓。

delay(3) 怎麼幾乎沒停？

單位是毫秒！3 秒 = 3000。

loop 圈數跑好快？

正常，一圈 = 表演完＋休息完。想慢就把 delay 加大。
