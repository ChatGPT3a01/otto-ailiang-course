CH16 · 校正與微調

真機站不直？trim 校正解決「模擬與現實的差距」

曾慶良 主任（阿亮老師）
2026

這一章要做什麼

工程師的必修課：對付現實

模擬器裡 OTTO 永遠站得直挺挺——現實不會。伺服馬達組裝時卡榫差一格、齒輪有微小誤差，真機常常歪頭、外八、走路偏一邊。這不是你程式寫錯，是硬體個體差異，解法叫 trim 校正。

🎁 本章小成品

你的 OTTO 找到專屬的 4 個校正值，站得直、走得正，而且校正值永久存進機器人。

trim 是什麼？

給每顆馬達一個「補償角度」

trim（微調值）＝告訴每顆馬達：「你的 0 度其實要多轉幾度才是真的正」。四顆馬達各一個值，通常在 -15 ～ +15 之間：

Otto.setTrims(左腿, 右腿, 左腳, 右腳);
Otto.setTrims(  0,   -3,    5,    2 );   // 每台機器人的數字都不同！
Otto.saveTrimsOnEEPROM();   // 存進機器人的永久記憶，斷電不會忘
🧠 EEPROM 是 Nano 上的小小永久記憶體。校正值存進去以後，之後上傳任何程式都自動生效，一勞永逸。
步驟 1 / 5

先觀察：哪裡歪？

上傳「立正程式」看真相
  • 用一鍵上傳頁傳 otto_robot.ino（CH04 骨架：開機立正就停）
  • OTTO 站好後，從正面和側面各看一次
  • 記下來：頭歪左還歪右？哪隻腳掌沒貼地？腿有沒有外八？
✅ 產出：一張「歪斜清單」，例如「右腳尖翹起、雙腿微外八」。
步驟 2 / 5

建立校正程式

在 Arduino IDE 貼這支小程式
// otto_trim.ino —— 校正專用小程式
#include <Otto9.h>
Otto9 Otto;

void setup() {
  Otto.init(2, 3, 4, 5, true, A6, 13, 8, 9);
  // ↓↓ 只改這一行的四個數字（左腿, 右腿, 左腳, 右腳）↓↓
  Otto.setTrims(0, 0, 0, 0);
  Otto.saveTrimsOnEEPROM();   // 存進永久記憶
  Otto.home();                // 用新校正值立正，馬上看效果
}

void loop() {
}

開 Arduino IDE（CH15 學的）→ 新檔案 → 貼上 → 存檔叫 otto_trim。

步驟 3 / 5

猜一個值，上傳看看

校正就是「真機版的迭代循環」
  • 照你的歪斜清單改數字。方向口訣：腳掌翹起來 → 那隻腳的值往下調（或反向）；腿外八 → 腿的值往內調。先從 ±5 猜起
  • 按上傳 → OTTO 用新值立正
  • 看結果：更直了？繼續微調；反而更歪？往反方向調
🔁 有沒有很熟悉？執行 → 觀察 → 修正 → 再執行——Loop 工程用在硬體上，一模一樣。每台 OTTO 的答案都不同，這是「你的機器人專屬密碼」。
步驟 4 / 5

走路驗證

站得直還要走得正
  • 站直後，上傳 otto_walk.ino（一鍵上傳即可）
  • 放地板走 10 步：有沒有明顯偏左/偏右？
  • 偏的話回步驟 3 再微調腿的 trim（±2 慢慢調）
✅ 預期：直線 10 步偏移不超過半個手掌。滿分不必強求——真實機器人有點個性很正常。
步驟 5 / 5

記錄你的專屬值

寫下來，以後都用得到
  • 把最終四個數字抄在筆記／貼紙貼機器人底部（例：0, -3, 5, 2）
  • 已 saveTrimsOnEEPROM → 之後上傳任何程式（含一鍵上傳）都自動套用
📚 覺得「猜數字 → 上傳 → 再猜」有點慢？下面幾頁教你更直覺的鍵盤即時校正（按一下鍵，馬達馬上動、馬上存），還有真機最容易踩到的兩個雷怎麼救。

更快的方法：鍵盤即時校正

按一下鍵，馬達馬上動、馬上存

不用一直改數字、重新上傳。這支程式讓你打開序列埠監控視窗，用鍵盤一格一格微調 4 顆馬達，每按一下就即時存進 EEPROM，邊按邊看 OTTO 站姿變化。

// otto_calibration.ino —— 鍵盤即時校正
#include <Otto9.h>
#include <EEPROM.h>
Otto9 Otto;
int YL=0, YR=0, RL=0, RR=0;   // 左腿 右腿 左腳 右腳 的微調值

void setup() {
  Otto.init(2, 3, 4, 5, true, A6, 13, 8, 9);
  Serial.begin(115200);
  YL=EEPROM.read(0); if(YL>128) YL-=256;   // 讀回上次存的值
  YR=EEPROM.read(1); if(YR>128) YR-=256;
  RL=EEPROM.read(2); if(RL>128) RL-=256;
  RR=EEPROM.read(3); if(RR>128) RR-=256;
  Otto.setTrims(YL, YR, RL, RR);
  Otto.home();
  Serial.println("a/z=左腿  s/x=左腳  k/m=右腿  j/n=右腳  f=試走  h=立正");
}

void loop() {
  if (Serial.available() > 0) {
    char c = Serial.read();
    if      (c=='a') YL++;  else if (c=='z') YL--;
    else if (c=='s') RL++;  else if (c=='x') RL--;
    else if (c=='k') YR++;  else if (c=='m') YR--;
    else if (c=='j') RR++;  else if (c=='n') RR--;
    else if (c=='f') { Otto.walk(1,1000,1); return; }   // 試走一步
    else if (c=='h') { Otto.home(); return; }            // 回立正
    else return;
    Otto.setTrims(YL, YR, RL, RR);
    Otto.saveTrimsOnEEPROM();   // 立刻存進永久記憶
    Otto.home();                // 用新值立正，馬上看效果
  }
}

鍵盤校正怎麼操作

打開序列埠監控視窗，開始微調
  • 把上面程式上傳到 OTTO（開發板設定跟 CH15 一樣：Nano ＋ Old Bootloader）
  • 上傳完，點 Arduino 右上角 🔍 序列埠監控視窗，右下角鮑率選 115200
  • 對照下表打字（打完按 Enter 送出），OTTO 會即時調整：
部位往一邊調往另一邊調
左腿az
左腳掌sx
右腿km
右腳掌jn
f = 試走一步看正不正　｜　h = 回立正姿勢
✅ 調到站直、腳掌貼地就完成，數值已自動存進機器人，不用另外按存檔。

⚠️ 除錯 1：一通電就「答答答」還發燙

馬達卡住了——先歸零、再裝腳
🔥 保命第一步：如果馬達一直答答答響不停、又摸起來發燙 → 馬上拔電！馬達卡住還硬撐，幾分鐘就會燒掉。

為什麼會這樣？馬達被程式命令回 90°，但這隻腳是趁馬達不在 90° 時鎖上的，它想回中間卻被卡在牆上到不了 → 硬頂 → 又響又燙。（新組裝第一次通電最常見）

正確順序「先歸零、再裝腳」——像時鐘先轉到 12 點，再把指針裝正：

  • 拔電後摸摸看哪顆馬達熱、看哪隻腳最歪
  • 鬆開那隻腳固定在馬達臂上的螺絲（讓腳能自由轉）
  • 插電，讓校正程式把馬達歸到 90°（鬆開後沒負載，答答答會停）
  • 趁馬達停在 90°，把腳擺正、再鎖回螺絲
  • 其他歪的腳重複做，最後用上一頁的鍵盤微調收尾

⚠️ 除錯 2：校正常見錯誤急救包

上傳／編譯卡關看這裡

編譯錯誤 Otto9.h: No such file？

Arduino 找不到函式庫。①確認 Libraries_V9 裡的資料夾（Otto9、Oscillator…）有放進 文件\Arduino\libraries。②裝好後要完全關閉 Arduino 再重開（它只在啟動時掃描）。③進階：檔案 → 偏好設定，確認「Sketchbook 位置」指向的就是放函式庫那個資料夾（有人的「文件」被導到別的磁碟，Arduino 會找錯地方）。

序列埠灰灰的不能選？

驅動沒裝（回 CH02）／線是純充電線／換一個 USB 孔。

上傳後馬達沒力、OTTO 一直重開？

供電不足！光靠電腦 USB 帶不動 4 顆馬達，接上電池盒再校正就穩了。

紅字 avrdude: stk500_getsync()？

處理器把 Old Bootloader ↔ 非 Old 對調再試、序列埠選對、換條線。
