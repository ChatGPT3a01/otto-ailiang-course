🎁 本章小成品

模擬器裡的 OTTO 聽你的程式「開機」：登登登 → 立正。之後每章都在這骨架上加本事。

Arduino 程式的形狀

每支 .ino 都有這三塊
區塊什麼時候跑放什麼
開頭（最上面）還沒開始跑載入函式庫、腳位設定（#define）
setup()開機時只跑一次接通零件、開機儀式
loop()跑完再跑一遍，永遠重複OTTO 平常要做的事
🏠 比喻：開頭＝進家門前先看地址門牌；setup()＝早上起床儀式（只做一次）；loop()＝上班的一天（每天重複）。

逐段看懂：開頭

不用背，看懂重點就好
#include <Otto9.h>   // 載入 OTTO 的能力包（原廠 V9 函式庫）
Otto9 Otto;          // 宣告一隻機器人，名字就叫 Otto

#define PIN_YL 2       // 左腿伺服馬達 → 腳位 2
#define PIN_YR 3       // 右腿伺服馬達 → 腳位 3
#define PIN_RL 4       // 左腳伺服馬達 → 腳位 4
#define PIN_RR 5       // 右腳伺服馬達 → 腳位 5
#define PIN_Trigger 8  // 超音波發射 → 腳位 8
#define PIN_Echo 9     // 超音波接收 → 腳位 9
#define PIN_Buzzer 13  // 蜂鳴器 → 腳位 13

眼熟嗎？這些數字就是 CH01 的「身體地圖」！#define 是幫數字取名字：之後寫 PIN_YL 就等於寫 2，但人看得懂。

逐段看懂：setup() 與 loop()

開機儀式＋永遠重複的心臟
void setup() {
  // 接通所有零件（參數順序是固定的，抄就對了）
  Otto.init(PIN_YL, PIN_YR, PIN_RL, PIN_RR, true, A6, PIN_Buzzer, PIN_Trigger, PIN_Echo);
  Otto.sing(S_connection);  // 唱開機音效：登登登！
  Otto.home();              // 立正站好
}

void loop() {
  // 之後每一章教的新本事都寫在這裡
}

Otto.init(...) 一行把 CH01 認識的所有零件接通；Otto.sing() 唱音效；Otto.home() 立正。loop() 現在是空的——這正是下一章要填的地方。

🔁 迭代任務

輪到你指揮：改造開機儀式

把你的程式貼給 AI，用自己的話請它修改，改完貼回模擬器驗證。挑戰任選（也可以自己想）：

  • ⭐ 開機音效換成超級開心（提示：S_superHappy）
  • ⭐⭐ 開機後多表演一個手勢（提示：Otto.playGesture(OttoHappy)）
  • ⭐⭐⭐ 開機唱歌之前先等 1 秒（提示：delay(1000)——想想要放在哪一行前面？）
💬 回饋句型參考：「這支程式開機會唱 S_connection，請幫我改成 ______，其他不要動。」

常見問題

卡住先看這

按執行沒反應？

看左下紅字。「找不到 void setup()」→ 程式沒貼完整，回生成器重新複製完整程式。

紅字「不認識的名稱」？

通常是 #define 那段沒貼到。開頭＋setup＋loop 三段都要在。

沒有聲音？

右上角聲音打勾；先用滑鼠點一下頁面再執行（瀏覽器規定）。

AI 生出來的跟老師版不一樣？

正常！AI 每次寫法可能小差異。課堂以生成器「採用」的版本為準，自由發揮留在迭代任務。
