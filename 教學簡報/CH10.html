🎁 本章小成品

一場屬於你的完整舞蹈秀：有開場亮相、有主秀高潮、有謝幕下台。這也是期末發表的雛形！

編舞的思考法：三幕式

先想劇本，再寫程式
🎬

第一幕 · 開場

音效吸睛 → 走位亮相 → 打招呼。讓觀眾知道「表演開始了」。

🌟

第二幕 · 主秀

最強舞步放這裡：滑步、交叉步、轉身。節奏由慢到快最抓人。

🎭

第三幕 · 謝幕

跳躍收尾 → 勝利手勢 → 溫馨音效 → 後退下台。有始有終。

✍️ 專業做法：先用中文把三幕劇本寫在紙上，再翻成提示詞給 AI——「先想清楚再開口」正是 Vibe Coding 的精髓。

程式裡的 // ===== 第一幕 ===== 註解就是「幕的分隔線」——寫長程式先分區，是好習慣（骨架地圖的精神）。

🔁 迭代任務（本章主菜！）

改造成「你的」舞蹈秀

這章的迭代任務就是正式創作。步驟：

  • 紙上寫你的三幕劇本（中文即可，例如「開場要神秘一點，用魔法滑步」）
  • 把老師範本＋你的劇本一起貼給 AI：「請照我的劇本改造這支程式」
  • 模擬器驗證 → 哪一幕不對就回饋 AI 修 → 直到滿意
  • 下載存檔：這支就是你的期末作品初稿！
⚠️ 規則：三幕結構要在、總長 20~40 秒、至少用到 3 種舞步＋2 種音效＋2 種手勢。其餘自由發揮！
